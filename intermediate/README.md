# Nivel intermedio

Aquí dejamos de aprender piezas aisladas y empezamos a diseñar programas que
pueden crecer sin convertirse en un cajón de cables misteriosos.

| # | Ejemplo | Aprenderás |
| --- | --- | --- |
| 01 | [Módulos y visibilidad](01-modules-and-visibility/) | separar responsabilidades y diseñar APIs |
| 02 | [Traits y genéricos](02-traits-and-generics/) | contratos compartidos y despacho estático |
| 03 | [Lifetimes](03-lifetimes/) | relacionar la validez de referencias |
| 04 | [Closures e iteradores](04-closures-and-iterators/) | pipelines perezosos y captura de contexto |
| 05 | [Errores propios](05-custom-errors/) | errores con fuente y propagación |
| 06 | [Estrategias de pruebas](06-testing-strategies/) | unitarias, integración y doc tests |
| 07 | [CLI](07-command-line-app/) | argumentos, salida y códigos de error |
| 08 | [Serde](08-serde-json/) | datos tipados desde y hacia JSON |
| 09 | [Threads](09-threads/) | paralelismo con memoria compartida segura |
| 10 | [Channels](10-channels/) | comunicar trabajo entre hilos |
| 11 | [Smart pointers](11-smart-pointers/) | `Box`, `Rc`, `RefCell` y propiedad interior |
| 12 | [Workspace](12-workspace-architecture/) | paquetes, dependencias de ruta y fronteras |

Sigue el orden: cada ejemplo puede ejecutarse solo, pero el vocabulario se apoya
en las lecciones anteriores. La independencia es técnica; el aprendizaje sigue
siendo una escalera, no un elevador con botones aleatorios.
